## 3. Deployment enablement

### 3.1 Self-bootstrapping vector index (code change)

On a fresh deploy `chroma_store/` does not exist (gitignored), so the
collection is empty and Ask-the-Data returns nothing. Fix: in the
Ask-the-Data submit handler in `app.py`, if the collection is empty, build
the index before answering:

```python
from rag.vector_store import get_collection
if get_collection().count() == 0:
    with st.spinner("Building knowledge index..."):
        from scripts.load_knowledge_base import main as build_index
        build_index()
```

This runs lazily (only on first question), so it does not slow dashboard
startup, and makes the app self-provisioning without committing the binary
store. Behavior is unchanged locally where the index already exists.

### 3.2 `.streamlit/config.toml`

Basic theme + server config (e.g. headless server settings, theme colors).
Committed so local and deployed runs share configuration.

### 3.3 Deployment docs (in README)

- **Hugging Face Spaces:** the required README YAML frontmatter snippet
  (`sdk: streamlit`, `sdk_version`, `app_file: app.py`, etc.) provided as a
  copy-paste block in the guide, plus push/setup steps. HF free tier (16GB)
  comfortably runs the RAG model.
- **Streamlit Community Cloud:** connect-repo steps, with an explicit warning
  that the ~1GB free-tier RAM may OOM when FLAN-T5 loads on the first
  question; recommend HF Spaces for the full RAG experience.
